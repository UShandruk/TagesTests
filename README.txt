# 1. О ПРОЕКТЕ
Данный проект выполнен в качестве тестового задания на должность Automation QA Инженер (Middle/Senior) в VisualStudio 2022.
В нем всего 3 теста на 4 кейса, но сделан он так, чтобы было удобно добавлять новые тесты.

# 2. ОТКРЫТИЕ ПРОЕКТА
Данный проект открывается в VisualStudio.
Существует два способа его открыть:
Способ №1. Двойной клик левой кнопки мыши по файлу AutotestSolution.sln
Способ №2. В верхней части окна VisualStudio нажать File -> Open -> "Project/Solution" -> выбрать файл AutotestSolution.sln -> Открыть

# 3. ЗАПУСК ТЕСТОВ
Для запуска какого-либо теста необходимо:
1. Выбрать нужный тест в окна TestExplorer (тесты пока запускаются по одному).
2. Нажать на кнопку "Play" в верхней части окна TestExplorer программы VisualStudio (зеленый треугольник)
Примечание: если окна TestExplorer нет, его можно открыть нажав в верхней части окна VisualStudio нажать View -> TestExplorer.

# 4. СТРУКТУРА ПРОЕКТА
AutoTestSolution\Pages                  - Папка с моделями страниц тестируемого веб-приложения //https://tages.ru
AutoTestSolution\Pages\PageAbstract.cs    - Модель страницы тестируемого веб-приложения (общие методы для всех страниц)
AutoTestSolution\Pages\PageMain.cs        - Модель главной страницы тестируемого веб-приложения

AutoTestSolution\Tests                  - Папка с тестами
AutoTestSolution\Tests\TestsMainPage.cs   - Тесты главной страницы тестируемого веб-приложения.
 !Внимание. Для успешного выполнения теста 2 (Кейс 2 - Проверить вызовы email и tel) необходимо:
  - чтобы на компьютере, на котором выполняется данный тест, были запущены
     приложение для отправки почты и приложение для совершения телефонных звонков.
  - чтобы их названия были указаны в AutoTestSolution\Settings.cs // переменные ProcessEmailName и ProcessPhoneName
 Это нужно для проверки работы ссылок в разделе Контакты.

AutoTestSolution\Utilites               - Папка с методами которые могут пригодиться в других тестах (общие методы для всех страниц)
AutoTestSolution\Utilites\Utilites.cs     - Методы, которые могут пригодиться в других тестах

AutoTestSolution\Utilites\Settings.cs     - Настройки, общие для всех тестов
